# Mindspar

Head-to-head thinking duels (18+). Ten questions across eight domains —
reasoning, math, verbal, general knowledge, science, patterns, history,
geography — scored on accuracy and speed. Duel a random player at your skill level, challenge a friend by email,
or take on domain-skilled bots (per subject, if you like). Your **Mindspar Score**
is an IQ-style number normalized within your age group (entertainment estimate,
not a clinical assessment — the app says so on the profile).

Two clients, one backend:

- **iOS** (SwiftUI, 17+) — this repo's Xcode project.
- **Web** (`web/`) — static, no build step, playable in any browser. See
  [web/README.md](web/README.md) for setup and free hosting options. The web
  client additionally has: **light/dark themes**, a **friends system** (add by
  email, accept/decline, challenge), **end-to-end encrypted 1:1 text chat**
  between friends (Web Crypto ECDH + AES-GCM; the server only ever stores
  ciphertext), **email verification** at signup, and a **30-minute idle
  sign-out**. These are web-only for now — see the iOS parity note below.

All 1,200 questions, bot personalities, names, and visuals are original — the game
shares a *genre* with titles like Brain Wars and QuizUp, but no content, assets,
or trademarks from them.

## Skill system

- **Elo rating** (start 1000, K=32) settles after every duel — bots have
  calibrated ratings, so bot matches move it honestly. Tiers: Novice → Adept →
  Scholar → Sage → Luminary. Rating *is* the level; there's no separate XP.
- **Matchmaking** claims the closest-rated lobby opponent (±200 band, stretches
  only if the lobby is empty).
- **Adaptive difficulty**: questions carry difficulty 1–3; decks are drawn toward
  your rating band (human matches use the average of both ratings, stored in the
  match doc).
- **Freshness + repeat cap**: a question you've seen in the last 7 days — or
  ever been served 3 times — is avoided while alternatives exist (bot matches;
  shared human decks can't do per-user avoidance).

## Running the iOS app

1. Xcode 15+ (free). Open `Mindspar.xcodeproj`, select your team, ⌘R.
2. **With no further setup the app runs fully offline**: local profile (still
   requires the 18+ DOB gate), all five bots, stats, and the Mindspar Score.
   Online play shows a friendly "needs Firebase" message.

The project is generated by XcodeGen from `project.yml` (`xcodegen generate` after
adding files). The Firebase iOS SDK is pulled via Swift Package Manager on first open.

## Enabling online multiplayer (free)

1. Create a project at console.firebase.google.com (free Spark plan is plenty).
2. Add an iOS app with bundle ID `com.rakesh.mindspar`, download
   **GoogleService-Info.plist**, and drop it into the `Mindspar/` folder
   (add it to the target in Xcode). The file is gitignored — never commit it.
3. In the console: **Authentication → Sign-in method → Email/Password → Enable**,
   and **Firestore Database → Create** (production mode).
4. Deploy the hardened rules: `firebase deploy --only firestore:rules`
   (they live in [`firestore.rules`](firestore.rules)). These enforce, server-side:
   email-verified accounts only; per-collection ownership; friends-only chat;
   and no bulk listing of the user table. Don't fall back to a blanket
   `allow read, write: if request.auth != null` — that was the old loophole.

The app detects the plist at launch and switches from `LocalBackend` to
`FirebaseBackend` automatically — no code changes. The web client reuses the
same Firebase project (add a *web app* in project settings; see web/README.md).

## Architecture

- `Models/` — domains, questions, profiles, stats, match state, Elo, difficulty
  ladder, scoring.
- `Services/Backend.swift` — the `Backend` protocol + offline `LocalBackend`.
- `Services/FirebaseBackend.swift` — Auth, Firestore matchmaking lobby, invites,
  live answer sync.
- `Services/DuelEngine.swift` — one duel: countdown, timers, scoring, opponent
  feed (bot timeline or Firestore stream — the view can't tell which).
- `Services/Bots.swift` — five bots as pure skill profiles (per-domain accuracy +
  speed range). No LLM, no cost.
- `Services/IQModel.swift` — Mindspar Score: accuracy/speed skill index → z-score
  against an age-group reference → mean-100/SD-15 scale. Unlocks after 16 answers.
- `Resources/questions.json` — 1,200 original questions (150 per domain), difficulty-tagged.
  **Generated**: edit `tools/generate_questions.py` and re-run it (writes both
  this file and `web/questions.js`). Don't hand-edit the outputs.
- `web/` — the browser client (own README).

## Known cross-platform gaps (fix before iOS↔web play)

- **Decks**: iOS derives human-match decks from a shared `seed` (SplitMix64);
  the web stores explicit `deckIds` in the match doc. iOS should adopt `deckIds`.
- **Identity**: iOS signup asks for a unique username; web asks for a display
  name + matches by email. Align iOS to name-based signup.
